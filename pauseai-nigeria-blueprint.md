## **Why pause AI?  /why-pause**

*This is the intellectual engine of the site. It must be credible enough for a policymaker and clear enough for a student with no AI background.*

### **Section 1 — Page hero**

* Headline: "Why the most powerful AI systems must be paused"

* Subhead: "A clear-eyed case for Nigerian citizens, researchers, and decision-makers"

### **Section 2 — The risks (3 pillars)**

* Pillar 1 — Existential risk: What leading researchers actually say. Cite the 2023 Statement on AI Risk. Hinton/Bengio quotes. The 10–20% probability framing.

* Pillar 2 — Loss of control: Nobody fully understands how frontier models work. No off-switch. Race dynamics between labs.

* Pillar 3 — Nigeria's specific exposure: Economic disruption to white-collar sectors (finance, law, healthcare). AI-enabled disinformation targeting Nigerian elections. Dependency on foreign AI systems with no local oversight.

### **Section 3 — Our proposal**

* "A temporary pause on training the most powerful general AI systems — until we can prove they're safe."

* What it is NOT: not a ban on all AI; not anti-technology; not permanent

* What it IS: a call for democratic oversight, safety guarantees, international treaty

* Link to global proposal: pauseai.info/proposal

### **Section 4 — Nigeria in context**

* NITDA's current AI policy stance — link to tracker

* What Nigeria's tech sector stands to gain or lose

* What other African nations (Kenya, South Africa, Rwanda) are doing on AI governance

* The case for Nigeria leading on this, not just reacting

### **Section 5 — Counter-arguments addressed**

* "AI will create jobs." — addressed with data on net displacement

* "We can regulate it later." — addressed with pace-of-progress argument

* "This is a Western concern." — addressed with Africa-specific risk data

* "Nigeria has bigger problems." — addressed with co-existing priorities framing

### **Section 6 — Further reading**

* Links to 5–8 curated external resources (Hinton interview, AI Risk statement, etc.)

* CTA: "Read our full resources list" → /resources

## **Stack summary**

| Framework | Astro (static site generator with optional server endpoints) |
| :---- | :---- |
| **Styling** | TailwindCSS with custom design tokens (tailwind.config.mjs) |
| **CMS** | Decap CMS — editor at /admin, config at public/admin/config.yml |
| **Hosting** | Netlify — free tier (100GB bandwidth, 300 build minutes/month) |
| **Forms** | Netlify Forms — free tier (100 submissions/month) |
| **Newsletter** | Brevo — free tier (300 emails/day, unlimited contacts) |
| **Analytics** | Plausible — free self-hosted OR cloud trial; script in BaseLayout.astro |
| **Version ctrl** | GitHub — all contributors fork, PR to main, squash-merge |
| **Fonts** | Saira Condensed 700 (display) \+ Inter (body) via Google Fonts |

## **File structure conventions**

* src/pages/ — one .astro file per route. Dynamic routes use \[slug\].astro.

* src/layouts/ — BaseLayout.astro wraps every page (navbar, footer, meta)

* src/components/ — reusable UI components (Card.astro, Badge.astro, etc.)

* src/content/ — Decap CMS collections (blog/, events/, policy-tracker/, team/)

* src/styles/global.css — design tokens, component classes, prose overrides

* public/ — static assets (images, favicon, admin/ for Decap)

* netlify/functions/ — serverless functions (newsletter, RSS proxy)

## **Decap CMS setup (to be completed before launch)**

* Create public/admin/config.yml with collections matching src/content/ schema

* Enable Netlify Identity in Netlify dashboard (free tier)

* Invite content editors via Netlify Identity — they log in at /admin

* Set backend: name: git-gateway in config.yml

## **Windows dev environment notes**

* Ensure Windows Developer Mode is ON (Settings → Privacy & security → For developers)

* Use pnpm as package manager (pnpm dev, pnpm build, pnpm preview)

* Node version: LTS (v20.x). Use .nvmrc if multiple Node versions installed.

* Line endings: set git config core.autocrlf false to prevent CRLF issues

* VS Code extensions: Astro, Tailwind CSS IntelliSense, Prettier
